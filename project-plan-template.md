# 개인 미니 프로젝트 기획서 (템플릿)

> Day 3 오전 기획 세션에서 작성. 한 장 분량, `[ ]` 를 채우세요.

## 1. 문제 (한 문장)
- 누구의 어떤 불편을 해결하나? → **[RA 및 Medical Affairs 담당자가 규제 정보와 최신 논문를 각각 찾아 정리하는 과정이 반복적이고 많은 시간이 소요]**

## 2. 사용자 · 성공 기준
- 사용자: [ RA(Regulatory Affairs) 및 Medical Affairs 실무자, 관련 직무를 준비하는 약학 전공 학생 ]
- 성공 기준: [ 약물명을 입력하면 최신 허가 정보와 주요 논문를 자동으로 검색·요약하여 1페이지 형태의 보고서를 생성한다 ]

## 3. 범위
- 할 것 (필수): [ - 약물명 입력, PubMed에서 주요 논문 검색, 논문 핵심 내용(목적, 결과, 안전성) 요약, FDA 허가정보 요약, 하나의 텍스트 보고서 생성 ]
- 안 할 것 (이번엔 제외): [ GUI 개발, 데이터베이스 구축, 실시간 웹 크롤링 고도화, 논문 전문 분석 및 통계 검증 ]

## 4. 산출물
- 결과 형태: [ 실행 가능한 Python 프로그램 + 약물 정보 요약 보고서(.txt 또는 .md) ]
- 필요한 입력·데이터: [ 약물명(예: Leqvio), PubMed 검색 결과, FDA 공개 허가 정보 ]

## 5. 도구·계획
- 사용할 도구: Claude Code (+ 필요 시 oh-my-claudecode)
- 대략 순서: [ 1) 약물명 입력받기 2) 논문 및 허가정보 검색 3) AI가 핵심 내용 요약 4) 허가정보와 논문 내용 통합 5) 최종 보고서 생성]
